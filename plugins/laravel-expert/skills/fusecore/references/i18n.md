# Internationalization (i18n)

## Overview

FuseCore supports multi-language applications with translations at both backend (Laravel) and frontend (React) levels.

## Supported Locales

| Locale | Language |
|--------|----------|
| `fr` | French (default) |
| `en` | English |
| `de` | German |
| `it` | Italian |
| `es` | Spanish |

## Backend Translations (Laravel)

### Location

```
FuseCore/{Module}/Resources/lang/
├── fr/
│   └── messages.php
├── en/
│   └── messages.php
└── ...
```

### Loading

Via `HasModule` trait:

```php
$this->loadModuleTranslations();
```

### Usage

```php
__('user::messages.welcome')
// Resolves: FuseCore/User/Resources/lang/{locale}/messages.php
```

### Namespacing

| Module | Namespace |
|--------|-----------|
| User | `user::` |
| Blog | `blog::` |
| Dashboard | `dashboard::` |

## Frontend Translations (React)

### Location

```
FuseCore/{Module}/Resources/React/i18n/locales/
├── fr/
│   ├── common.json
│   └── auth.json
├── en/
│   ├── common.json
│   └── auth.json
└── ...
```

### File Structure

```json
// auth.json
{
  "login": {
    "title": "Connexion",
    "email": "Adresse email",
    "password": "Mot de passe",
    "submit": "Se connecter"
  }
}
```

### Usage in React

```typescript
import { useTranslation } from 'react-i18next';

const { t } = useTranslation('auth');

<h1>{t('login.title')}</h1>
```

## Configuration

### fusecore.php

| Setting | Default | Purpose |
|---------|---------|---------|
| `i18n.enabled` | true | Enable i18n |
| `i18n.default_locale` | `fr` | Default language |
| `i18n.fallback_locale` | `en` | Fallback if missing |
| `i18n.supported_locales` | `[fr,en,de,it,es]` | Available locales |

## Translation Keys

### Naming Convention

| Pattern | Example |
|---------|---------|
| `{section}.{key}` | `login.title` |
| `{section}.{subsection}.{key}` | `profile.settings.privacy` |

### Best Practices

| Practice | Reason |
|----------|--------|
| Lowercase keys | Consistency |
| Dot notation | Nesting |
| Descriptive names | Clarity |

## Pluralization

### Laravel

```php
'items' => '{0} Aucun élément|{1} Un élément|[2,*] :count éléments'
```

### React (i18next)

```json
{
  "items": "{{count}} élément",
  "items_plural": "{{count}} éléments"
}
```

## Variables

### Laravel

```php
// messages.php
'welcome' => 'Bienvenue, :name !'

// Usage
__('user::messages.welcome', ['name' => 'Jean'])
```

### React

```json
{
  "welcome": "Bienvenue, {{name}} !"
}
```

```typescript
t('welcome', { name: 'Jean' })
```

## Module Organization

### Per-Feature Files

```
i18n/locales/fr/
├── common.json      # Shared strings
├── auth.json        # Authentication
├── profile.json     # Profile page
└── errors.json      # Error messages
```

## Switching Locales

### Backend

```php
app()->setLocale('en');
```

### Frontend

```typescript
import { useTranslation } from 'react-i18next';

const { i18n } = useTranslation();
i18n.changeLanguage('en');
```

## Best Practices

1. **Namespace translations** - Prevent conflicts
2. **One file per feature** - Easy maintenance
3. **Use fallback** - English as fallback
4. **Extract all strings** - No hardcoded text
5. **Keep consistent** - Same key structure across locales

## Common Patterns

### Form Labels

```json
{
  "form": {
    "email": "Adresse email",
    "password": "Mot de passe",
    "submit": "Envoyer"
  }
}
```

### Error Messages

```json
{
  "errors": {
    "required": "Ce champ est obligatoire",
    "email": "Adresse email invalide",
    "min": "Minimum {{min}} caractères"
  }
}
```

### Notifications

```json
{
  "notifications": {
    "success": "Opération réussie",
    "error": "Une erreur est survenue"
  }
}
```
