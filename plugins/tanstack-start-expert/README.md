# fuse-tanstack-start

Expert TanStack Start development — the full-stack React framework (Release Candidate) with file-based routing, `createServerFn` server functions, selective SSR, middleware, server routes, and Nitro-powered deployment.

## Agent

| Agent | Description |
|-------|-------------|
| **tanstack-start-expert** | Expert TanStack Start full-stack developer (React, RC) |

## Skills

| Skill | Description |
|-------|-------------|
| solid-tanstack-start | SOLID principles for TanStack Start with modular architecture (files < 100 lines) |
| start-core | Framework fundamentals, `tanstackStart()` Vite plugin, project layout, versions |
| start-server-functions | `createServerFn`, `.validator()`, `.handler()`, GET/POST methods |
| start-middleware | `createMiddleware` for request and server-function middleware chains |
| start-execution-model | Isomorphic loaders vs server-only boundaries, where code runs |
| start-server-routes | API / server routes via `server.handlers` (GET/POST/PUT/DELETE) |
| start-routing-data | File-based routing, loaders, search params, data loading patterns |
| start-auth | Sessions, login/logout server functions, protecting data and functions |
| start-deployment | Nitro targets — Vercel, Cloudflare, Netlify, Node standalone |

## Architecture

```
src/
├── routes/
│   ├── __root.tsx
│   ├── index.tsx
│   └── posts/
│       └── $postId.tsx
├── router.tsx           # createRouter + route tree
├── routeTree.gen.ts     # generated by the router plugin — never edit by hand
├── server.ts            # server entry
└── styles.css
vite.config.ts           # tanstackStart() + viteReact() (+ Nitro/host plugin)
```

## Installation

```bash
/plugin install fuse-tanstack-start
```

## Usage

The agent activates when a TanStack Start project is detected:
- `@tanstack/react-start` in `package.json`
- `tanstackStart()` from `@tanstack/react-start/plugin/vite` in `vite.config.ts`
- `src/routes/` directory with a generated `routeTree.gen.ts`

For plain React SPAs without Start use `react-expert`; for Next.js use `nextjs-expert`.
